什麼是 AI Agent？

Agent ≠ 聊天機器人。它是一個能自主觀察、思考、行動的智慧體

👁️ Observe 觀察
接收環境資訊：API 回傳、檔案內容、網頁狀態、使用者輸入
🧠 Think 思考
LLM 推理：拆解任務、規劃步驟、選擇工具、生成回應
⚙️ Act 行動
執行工具：搜尋網路、讀寫檔案、操作瀏覽器、執行程式碼
Agent = LLM + 工具調用 + 記憶 + 規劃

基礎 進階

階段 1-2：先跑通一個 Agent

🎯 目標：寫出 50-150 行的最小 Agent，能調用工具並返回結果

📡 Function Calling
OpenAI / Claude / Gemini 都支援結構化工具調用，模型會回傳 JSON 格式的 tool call
🔄 Agent Loop
Observe → Think → Act → Observe 循環，加上最大步數、逾時、錯誤處理
🧰 定義工具
把搜尋、計算機、讀檔案包裝成工具函數，讓 Agent 按需呼叫
📖 學習資源
OpenAI Function Calling、Claude Tool Use、Gemini API Function Calling

階段 3-4：工具與多 Agent 協作

🔍 RAG 檢索增強
Chunk → Embed → Retrieve → Answer with citations
🛠️ 工具生態
搜尋引擎、資料庫、瀏覽器、程式碼執行器、檔案系統
👥 Multi-Agent
Supervisor / Graph 協作，避免 agent 閒聊，定義職責邊界
🧠 記憶管理
短期上下文 vs 會話記憶 vs 長期記憶；mem0 / Letta

階段 5：Skill 與通訊協議

現代 Agent 的能力不只來自模型，更來自可復用的能力包與標準化通訊

📦 Skills
可復用的流程知識包：SKILL.md + 腳本 + 模板，告訴 Agent 如何完成一類任務
🔌 MCP (Model Context Protocol)
標準化 Agent 連接外部工具、資料源和服務的方式
🤝 A2A (Agent-to-Agent)
讓不同 Agent 之間發現、通訊和協作的開放協議
💻 ACP (Agent Client Protocol)
編輯器、終端、IDE 與 Agent 之間的統一介面

階段 6-7：瀏覽器與評測

🌐 瀏覽器 Agent
  • browser-use / Playwright
  • 頁面觀察、點擊、提取資訊
  • 安全限制：不登入敏感帳號
  • 截圖 + DOM + 動作日誌
  • 失敗恢復機制
+
📊 評測 & 安全
  • 固定測試集，不只靠 demo
  • 成功率 / 成本 / 延遲追蹤
  • 危險工具加人工確認
  • Prompt Injection 防護
  • 回歸測試防止退化
